Dymo Printer Framework for .NET 4.5+ and barcodes

January 1, 0001

For a certain work project, I was uanble to print barcodes using C# and the dynmo .NET library. I could print text just fine. I just could not print barcodes for whatever reason. It was ALWAYS blank.
This is out here so if anyone is looking for an answer, here it is.
I googled and googled and the dymo forums are closed and there is NO Google cache for the forums either. I was stuck.
I then changed the project to .NET 3.5 and BAM it printed fine…. If anyone knows of a way to fix it please let me know at: drewskiwooskie[at]gmail[dot]com.
You cannot just call a .NET 3.5 .dll from a .NET 4.5 program, because it runs it in .NET 4.5, so it still will not print the barcode.
I write a .NET cli app that takes command line parameters which then prints to the printer specified in the cli. I then shell out to it from the .NET 4.5 program.
It works but I feel so dirty.
